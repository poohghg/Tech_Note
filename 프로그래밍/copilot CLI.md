
### 1. Copilot Instructions (코파일럿 기본 지침)

코파일럿이 코드를 생성하거나 자동 완성할 때 **항상 기본적으로 따라야 하는 전반적인 규칙**을 설정한다.

- **활용 예시:** * "항상 TypeScript를 사용하고 타입 정의를 엄격하게 해줘."
    - "변수명은 항상 camelCase로 작성해 줘."
    - "React 컴포넌트는 함수형으로 만들고 화살표 함수를 사용해 줘."

### 2. Instruction Files (파일별/작업별 지침 파일)

특정 파일 확장자나 특정 작업 유형에 맞춰 세부적인 규칙을 적어둔 `.instructions.md` 파일을 설정한다. 프로젝트 내에 이 파일을 만들어두면, 코파일럿이 해당 파일을 편집할 때 그 안의 규칙을 참고한다.
- **활용 예시:** * `UI.instructions.md` 파일을 만들어 "버튼 컴포넌트를 만들 때는 항상 Tailwind CSS 클래스를 사용해 줘"라고 명시하는 등, 전역 설정보다 세밀한 맥락을 제공할 때 씁니다.

### 3. Git Commit Instructions (커밋 메시지 지침)

코파일럿이 코드 변경 사항을 분석하여 **Git 커밋 메시지를 자동 생성해 줄 때의 규칙**을 설정합니다.

- **활용 예시:**
    
    - "커밋 메시지는 항상 한국어로 작성해 줘."
        
    - "Conventional Commits 형식(예: `feat:`, `fix:`, `docs:`)을 무조건 지켜서 작성해 줘."
        
    - "제목은 50자 이내로, 본문은 상세한 변경 이유를 포함해 줘."
        

### 4. AGENTS.md (에이전트 컨텍스트 파일)

채팅 창(Copilot Chat)에서 대화할 때, 프로젝트의 전반적인 아키텍처나 주요 기술 스택, 도메인 지식 등을 코파일럿에게 주입하기 위해 사용하는 파일입니다. 루트 디렉토리에 `AGENTS.md`를 만들어두면 챗봇이 그 내용을 배경지식으로 깔고 대화에 임합니다.

- **Use AGENTS.md file (체크박스):** 체크하면 채팅 요청을 보낼 때마다 프로젝트에 있는 `AGENTS.md` 파일의 내용을 컨텍스트로 함께 전송합니다.
    
- **Use nested AGENTS.md files (체크박스 - 실험적 기능):** 프로젝트 최상단뿐만 아니라 하위 폴더들에 있는 여러 개의 `AGENTS.md` 파일들도 모두 읽어서 컨텍스트로 사용합니다. 모노레포(Monorepo)처럼 폴더별로 환경이 다를 때 유용합니다.
