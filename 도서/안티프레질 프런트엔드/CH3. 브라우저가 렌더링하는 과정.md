> 웹 페이지의 도메인 주소를 브라우저 주소창에 입력한 후 웹 페이지가 화면에 표시되기까지의 과정

### 1.도메인 주소에서 웹 서버 구조 

#### DNS 리졸브

> 브라우저는 도메인 주소를 IP 주소로 변환하기 위해 DNS(Domain Name System) 리졸버를 사용한다. DNS 리졸버는 도메인 이름을 해당하는 IP 주소로 매핑하는 역할을 한다. 브라우저는 먼저 로컬 DNS 캐시를 확인하고, 캐시에 없으면 ISP(인터넷 서비스 제공자)의 DNS 서버에 질의하여 IP 주소를 얻는다.

> HTTP가 이해하는 주소는 도메인이 아닌 IP 주소이다. 사람인 이해하기 쉬운 형태인 도메인 주소를 프로토콜이 이해하는 형태인 IP 주소로 변환해야 브라우저와 웹 서버가 서로 통신하며, 브라우저는 사용자가 입력한 도메인 주소를 IP 주소로 변환해서 웹 페이지 정보를 가진 웹 서버로 요청을 보낸다.

#### DNS의 구성

> DNS는 도메인 네임 시스템의 약자로 인터넷에서 도메인 이름을 IP 주소로 변환하는 시스템이다. DNS는 분산된 데이터베이스 구조로 구성되어 있으며, 여러 계층의 서버들이 협력하여 도메인 이름을 관리하고 변환한다.

![[Pasted image 20251203191942.png]]

- 최상위 도메인(TLD, Top-Level Domain): 최상위 도메인은 도메인 이름의 가장 오른쪽 부분을 의미하며, 일반적으로 국가 코드나 일반적인 용도를 나타낸다. 예를 들어, .com, .org, .net, .kr 등이 있다.
- 차상위 도메인(SLD, Second-Level Domain): 차상위 도메인은 최상위 도메인 바로 왼쪽에 위치한 부분으로, 조직이나 회사의 이름을 나타낸다. 예를 들어, example.com에서 example이 차상위 도메인이다. 차상위 도메인은 TLD(Top-Level Domain) 아래에 위치하며, TLD 내에서 유일 무이해야 하며 일반적으로 웹 페이지를 만들고 도메인을 등록한다고 할 때는 SLD와 TLD를 결졍하여 도메인 등록 기관에 해당 도메인과 웹페이지의 IP 주소를 연결하는 것을 의미한다.
- 서브 도메인(Subdomain): 서브 도메인은 차상위 도메인 왼쪽에 위치한 부분으로, 조직 내의 특정 부서나 서비스를 나타낼 수 있다. 예를 들어, blog.example.com에서 blog가 서브 도메인이다. 특정 도메인을 소유하고 있다면 추가적인 여러 계층의 도메인을 만들 수 있다. 하위 도메인을 사용함으로써 조직이 하나의 SLD 아래에 여러 웹 페이지를 수축하고 관리하며, 각각의 하위 도메인이 서로 다른 IP주소나
