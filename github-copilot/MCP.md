> GitHub Copilot의 내장 도구(코드 검색, 파일 편집, 터미널 접근)는 다양한 작업을 지원한다.
> 하지만 실제 워크플로우에서는 데이터베이스,. 클라우드 API, 모니터링 대시보드 또는 내부 서비스와 같은 외부 시스템에 대한 접근이 필요한 경우가 많다. 바로 이러한 상황에서 MCP서버가 필요하다.

### MCP란?

모델 컨테스트 프로토콜(MCP)은 AI비서를 외부 데이터 소스 및 도구에 연결하기 위한 개발형 표준이다.
MCP서버는 대화 중에 호출 할 수 있는 기능을 노출하는 경량 프로세스이다. MCP서버는 AI에이전트가 외부 시스템과 상호 작용할 수 있도록 하는 인터페이스 역할을 한다. AI에이전트는 MCP서버에 HTTP요청을 보내고, MCP서버는 요청된 기능을 실행하고 결과를 반환한다. 

```
GitHub Copilot  ←→  MCP Server  ←→  External System                     (bridge)                      (bridge)      (database, API, etc.)
```

#### 주요 특징

- MCP는 GitHub Copilot에 특화된 프로토콜이 아니라 다양한 AI 도구에서 작동하는 개방형 프로토콜이다.
- 서버는 로컬 컴퓨터 또는 컨테이너에서 실행된다.
- 각 서버는 정의된 입력과 출력을 가진 하나 이상의 도구를 제공한다.
- 상담원과 사용자는 대화 중에 자연스럽게 MCP 도구를 호출할 수 있다.

GitHub Copilot은 항상 사용할 수 있는 여러 가지 **내장 도구를** 제공한다 .

|내장 도구|이 제품의 기능은 무엇인가요?|
|---|---|
|`codebase`|저장소 전체의 코드를 검색하고 분석합니다.|
|`terminal`|통합 터미널에서 셸 명령을 실행합니다.|
|`edit`|워크스페이스에서 파일을 생성하고 수정합니다.|
|`fetch`|URL에 HTTP 요청을 보냅니다.|
|`search`|워크스페이스 파일 전체에서 검색|
|`github`|GitHub API와 상호 작용하기|

**MCP 도구는** 외부 기능을 통해 이를 확장합니다.

|MCP 서버 예제|이 제품이 추가하는 기능은 무엇인가요?|
|---|---|
|포스트그레스SQL 서버|데이터베이스를 쿼리하고, 스키마를 검사하고, 쿼리 계획을 분석합니다.|
|도커 서버|컨테이너 관리, 로그 검사, 서비스 배포|
|센트리 서버|오류 보고서를 가져오고 충돌 데이터를 분석합니다.|
|Figma 서버|디자인 토큰과 구성 요소 사양을 읽으세요.|
