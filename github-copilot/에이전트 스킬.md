에이전트 스킬(agent skills)은 현재 여러 AI 에이전트 제품이 채택하는 강력한 오픈 표준으로 발전했다.
과거 클로드 코드(claude code)는 에이전트의 행동을 확장하기 위해 **사용자 지정 명령어(Custom Commands)**라는 방식을 사용했다. `.claude/commands/` 폴더 내에 단일 마크다운(.md) 파일 형태로 프롬프트 템플릿을 저장하는 단순한 형태였다. 이는 다음과 같은 한계점이 있었다.

- 사용자가 항상 슬래시(/) 명령어를 통해 수동으로만 호출해야 했으며, AI가 스스로 판단하여 자동 실행하는 기능(Auto-triggering)이 없었다.
- 명령어가 실행될 때마다 전체 프롬프트가 한 번에 로드되어 AI의 컨텍스트 윈도우(토큰)를 크게 낭비했다.
- 스크립트를 번들링하거나 복잡한 참조 문서를 분리해서 관리할 수 없다는 구조적 한계가 있었다.

사용자 지정 명령어의 한계를 극복하기 위해 앤스로픽(anthropic)은 새로운 방식인 에이전트 스킬을 개발했고, 오픈 표준으로 공개했다. 단일 파일 방식에서 벗어나, `SKILL.md` 파일을 중심으로 한 디렉토리 구조를 새롭게 채택했다. 이를 통해 실행 스크립트(scripts/)와 참조 문서(references/)를 분리하여 체계적인 관리가 가능해졌다. 사용자 의도를 파악해 0.1초 만에 알아서 스킬을 활성화하는 자동 호출(Model-invoked) 기능이 도입되었다.

필요한 순간에만 문서를 읽어들이는 **점진적 정보 노출(Progressive Disclosure) 메커니즘**을 적용하여 컨텍스트 토큰 소비를 극적으로 줄였다. 점진적인 정보 노출 메커니즘은 에이전트가 한정된 컨텍스트 윈도우(토큰)를 극도로 효율적으로 관리하기 위해 채택한 핵심 설계 원칙이다. 이 메커니즘은 수십 개의 스킬이 설치되어 있더라도 에이전트가 모든 내용을 처음부터 다 읽지 않게 막아주며, **발견(Discovery), 활성화(Activation), 실행(Execution)**의 3단계로 정보를 나누어 필요할 때만 로드한다.

- 발견(카탈로그 로드)
	- 세션이 시작될 때, 에이전트는 사용 가능한 모든 스킬의 이름과 간단한 설명이 담긴 카탈로그를 로드한다. 이 단계에서는 스킬의 상세 내용이나 실행 방법은 포함되지 않는다.
	- 아 단계에서 스킬당 약 10~20 토큰 정도의 최소한의 정보만 소비한다. 에이전트는 어떤 스킬이 존재하고 언제 사용할 수 있는지에 대한 개요만 갖게 된다.
- 활성화(지시사항 로드)
	- 사용자의 요청이 특정 스킬의 설명과 일치하여 에이전트가 해당 스킬이 필요하다고 판단하면, 에이전트는 그 스킬의 지시사항(instructions)을 로드한다. 지시사항은 스킬을 어떻게 활성화하고 사용할 수 있는지에 대한 구체적인 정보가 담긴 부분이다.
- 실행 및 자원 활용(리소스 온디멘드 로드)
	- 스킬이 활성화된 후, 에이전트가 스킬을 실행하기 위해 필요한 리소스(예: 외부 API 호출, 참조 문서 등)를 온디멘드 방식으로 로드한다. 
	- 에이전트는 이 추가 리소스들을 한 번에 로드하지 않고, 지시사항을 수행하는 과정에서 특정 파일의 정보나 스크립트 실행 결과가 정말로 필요한 순간에만 해당 리소스를 불러온다. 이를 통해 불필요한 토큰 소비를 방지하고, 에이전트가 더 많은 스킬을 효율적으로 관리할 수 있게 된다.
